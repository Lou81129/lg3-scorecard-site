What’s New – Build 92

Released: June 17, 2026 · Built from v91.2 baseline

New Features

Static Match Play Team Labels Removed

The hardcoded TEAM A and TEAM B labels that appeared above the Player 1/2 and Player 3/4 rows in every Match Play scoring grid have been permanently removed.

The problem: These labels were baked into the Match Play HTML at build time and always read “TEAM A” and “TEAM B” — even when the actual matchup was C vs D, E vs F, or G vs H. This made the card incorrect for 6 out of 8 possible pairings.

What was removed:

  • <span class="mhf-lbl">Team A</span> — static Match Header label
  • <span class="mhf-lbl">Team B</span> — static Match Header label

What was preserved:

  • Team letter dropdowns (A–H) — fully dynamic
  • Player name display spans — dynamically populated
  • mpSyncLabels() — continues to correctly generate “Team C · Best Ball,” “Team D · Best Ball,” etc.
  • All Match Play grid structure, MATCH row, and Best Ball rows

Game Selector Dropdown Disabled in Match Play Mode

The Select Game dropdown in the Setup Panel is now automatically disabled whenever Match Play mode is active.

Behavior in Match Play:

  • Dropdown disabled (non-interactive)
  • Greyed out — opacity: 0.45
  • Cursor changes to not-allowed
  • Hover tooltip: “Game selection applies to Stroke Play only.”

Behavior when switching back to Stroke Play:

  • Dropdown fully re-enabled
  • Normal opacity restored
  • Cursor returns to pointer
  • Tooltip cleared

What was NOT changed:

  • All Game Profile logic
  • All Stroke Play game behavior
  • 15/20 Balls Stableford layout
  • Switching between modes retains all player data
