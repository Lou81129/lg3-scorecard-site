What's New – Build 95

Released: July 20, 2026 · Built from v94.10 baseline

Internal Development Releases (v95.1 – v95.8)

These versions covered internal development and refinement of the HCP dropdown sync system for Scramble and Chapman team cards, Chapman game selection persistence, and foundational groundwork for the Par 5 shade toggle. No user-facing feature changes were released in these versions.

Added a Par 5 Shade Toggle button to the card control bar. Allows switching column shading from Par 3 holes (amber) to Par 5 holes (blue) for quick visual reference during play.

  • New Shade: Par 3 / Shade: Par 5 toggle button added to the button row
  • Added CSS variables --par5 (blue #d4eaf7) and --par5bw (grey #d8d8d8)
  • Button label and color update to reflect the active shade mode
  • Par 5 shading prints correctly in both Color and B&W modes

Par 5 Shade Toggle — Full CSS Class System (v95.10)

Complete CSS and JavaScript implementation for the Par 5 shade toggle introduced in v95.9.

  • Added body.p5-shade class — toggles Par 3 amber off and Par 5 blue on across all cell types
  • Added .p5c cell class to all Par 5 columns in buildTable
  • Added isP5() helper function and PAR5 / P5_MODE global variables
  • Chapman and Scramble team cards rebuild correctly when shade mode is toggled mid-session
  • B&W theme shades Par 5 columns in grey when toggle is active

Quota Game — Player 4 Label Fix (v95.21)

Fixed a bug where the Quota Game scoring row label for Player 4 failed to display the quota number unless the user tabbed away to a non-existent next field after data entry.

  • Root cause: The quota label refresh was not correctly wired into calcHcp(). Labels updated on player name input only — not when GHIN index or tee data was entered.
  • Fix: refreshPlayerNameGameRows() now fires at the end of calcHcp() for all four players — immediately on GHIN or tee entry, no tab-away required
  • Player 4 quota label now calculates correctly in all entry sequences

Quota Game — Weekly Game Notes Auto-Fill (v95.22)

When Quota Game is selected, the Weekly Game Notes block on the back panel automatically populates with the standard Quota payout rules.

Auto-Fill Content

  • Only golfers who finish over their quota earn shares of the pot
  • Your "+" number = your shares — Beat quota by +3 → 3 shares
  • Finish at 0 or negative → 0 shares
  • Total shares = sum of all positive "+" numbers
  • Payout per share:  Pot ÷ Total Shares
  • Golfer payout:  Shares × Payout per Share
  • If no golfer beats quota, all buy-ins are returned for that week

Smart Fill Behavior

  • Auto-fills only if the notes block is currently empty — never overwrites custom entries
  • Block remains fully editable after auto-fill — modify, add to, or clear as needed each week
  • Switching away from Quota Game auto-clears the default payout text
  • User-edited content is always preserved when switching between game formats
  • Font and size match the existing rules block (8pt Arial Narrow)
