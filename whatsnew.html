What’s New – Build 91.2

Released June 16, 2026 · Built from v91.1 baseline

New Features

15/20 Balls Stableford game type

Added as a selectable option in the Game dropdown. This format produces the Stableford layout with Points rows and a 3‑row game section.

Stableford Points row

A thin, light‑green shaded row (18 boxes) now appears beneath each player’s score row when 15/20 Balls Stableford is selected.

3‑row Stableford game section

The bottom game area now shows: # Scores · Total Points per Hole · Cumulative Points when Stableford is active.

Bug Fixes

Rogue in‑card GAME dropdown

Removed the entire rsel_l / rsel_r DOM block and all related CSS from buildTable().

Game dropdown defaulting incorrectly

The game selector no longer defaults to Stableford. activeGameId is now hardcoded to none on startup.

Game selection persisting between sessions

Removed localStorage.setItem('lg3_game'). Game choice now resets on every load as intended.

Dead sync code in applyGameProfile()

Removed an orphaned cgs.value = gameId reference that no longer applied to the current UI.

Grey body background inside iframe

Changed the body background from #ccc to pure white to eliminate the visible grey border when embedded.

Redundant Export PDF button

Removed the button and its unused initPdfExport() code. Print Card already saves as PDF.

Unchanged / Preserved

  • All 8 standard game profiles (Arizona Shuffle, Montana Mayhem, One Gross/Two Low Net, etc.)
  • Course Profiles – Tampa Palms G&CC and Hunter’s Green
  • All 4‑card save and sync systems
  • Print layout (Color and B&W modes)
  • Manual tee time inputs
  • HG Blue/White tee rating 70.8 / slope 130
  • “– Select Game –” placeholder defaults on every fresh load
